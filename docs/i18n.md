# 다국어 지원 (i18n) 가이드

## 개요
`expo-localization`과 `i18n-js`를 사용해서 한국어/영어를 지원해요.
앱 시작 시 기기 언어를 자동으로 감지하고, 사용자가 앱 내에서 언어를 변경할 수 있어요.

## 폴더 구조
```
src/
├── i18n/
│   ├── index.ts   # i18n 초기화 및 t(), setLocale(), getLocale() export
│   ├── ko.ts      # 한국어
│   └── en.ts      # 영어
├── hooks/
│   └── useTranslation.ts  # t() 함수 + 언어 변경 감지 훅
└── store/
    └── languageStore.ts   # 언어 상태 관리 (AsyncStorage 저장)
```

## 언어 파일 구조
화면/기능 단위로 키를 관리해요:
```typescript
// ko.ts, en.ts 공통 구조
export default {
  auth: {          // 인증 관련 (로그인 화면)
    title: '...',
  },
  home: {          // 홈 화면
    profile: '...',
  },
  // 새 화면 추가 시 여기에 추가
};
```

## 사용 방법

### 화면/컴포넌트에서 사용
```typescript
import { useTranslation } from '@/hooks/useTranslation';

export default function MyScreen() {
  const { t } = useTranslation();

  return <Text>{t('auth.title')}</Text>;
}
```

### 변수가 포함된 문장
```typescript
// ko.ts
welcome: '{{name}}님 환영합니다'
// en.ts
welcome: 'Welcome, {{name}}'

// 사용
t('home.welcome', { name: profile.name })
```

### 언어 변경
```typescript
import { useLanguageStore } from '@/store';

const { language, setLanguage } = useLanguageStore();

// 한국어로 변경
setLanguage('ko');

// 영어로 변경
setLanguage('en');
```

## 새 화면 추가 시

1. `ko.ts`와 `en.ts`에 새 화면의 키 추가
```typescript
// ko.ts
myScreen: {
  title: '내 화면',
  description: '설명',
},

// en.ts
myScreen: {
  title: 'My Screen',
  description: 'Description',
},
```

2. 화면에서 `useTranslation` 사용
```typescript
const { t } = useTranslation();
<Text>{t('myScreen.title')}</Text>
```

> ⚠️ `ko.ts`와 `en.ts` 두 파일 모두에 키를 추가해야 해요.
> 한쪽에만 있으면 개발 환경에서 콘솔 경고가 출력돼요.

## 다국어가 필요 없는 경우
한국어만 사용하는 앱이라면 아래를 삭제하고 텍스트를 직접 하드코딩하면 돼요:
- `src/i18n/` 폴더 전체
- `src/hooks/useTranslation.ts`
- `src/store/languageStore.ts`
- `src/store/index.ts`에서 `useLanguageStore` export 제거
- `npm uninstall i18n-js expo-localization`
